Ось оновлений технічний опис та структура проєкту:

1. Календар активності (Activity Heatmap)
Механіка: Календар зчитує дати створення (createdAt) та останнього редагування (updatedAt) для всіх сутностей.

Візуалізація: Кожен квадрат має рівень "інтенсивності" (наприклад, 4 відтінки основного кольору). Якщо в один день пост був створений, а проєкт — відредагований, цей день отримує вищий бал активності.

Angular реалізація: Можна використати бібліотеку ngx-charts або створити кастомний SVG-компонент, який генерує сітку за поточний рік.

2. Секція «Блог» (Вертикальна стрічка)
Навігація: Класичний вертикальний скрол.

Компоненти: PostCardComponent для прев'ю та PostDetailComponent для повного перегляду.

Поля даних: Назва, головне фото, короткий опис, повний текст (з підтримкою Markdown або HTML), дата, список авторів (простий масив рядків), хештеги.

3. Секція «Проєкти» (Горизонтальний скрол)
Візуальний стиль: Сторінка працює за принципом горизонтальної галереї. Кожна картка проєкту займає значну частину екрана.

Реалізація скролу: Використання CSS-властивостей overflow-x: auto та scroll-snap-type: x mandatory для плавної фіксації карток при гортанні.

Особливість: При натисканні на проєкт відкривається детальна сторінка з галереєю (масив додаткових фото).

4. Система фільтрації та пошуку
Глобальний фільтр: Оскільки ви підтвердили необхідність фільтрації, натискання на хештег у будь-якому розділі (Блог чи Проєкти) має вести на сторінку результатів, де контент обох типів згрупований за цим тегом.

Angular Router: Використання Query Parameters (наприклад, ?tag=web-dev) для відображення відфільтрованого контенту.

5. Адмін-панель
Форми: Використання ReactiveForms для створення/редагування.

Поле авторів: Простий input, де імена додаються через кому або як теги (chip-input), що зберігаються в базі як масив рядків.
